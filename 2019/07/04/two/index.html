面向对象思想

主要是抽象，继承，封装，多态。

抽象是将一个现实事物进行抽象化，将其的特征和行为抽象为属性和方法，完成抽象后只需要关注对象的属性和方法就可以。

继承是对已有类的属性和行为的扩展，提供继承的是父类，继承的叫子类，子类可以重写父类方法，父类可以引用子类对象子类可以指向父类方法。

封装是将自定义接口让自身的属性和行为分装，可公开可隐藏，外接只要通过对外的接口操作对象类。

多态是通过继承对父类方法进行重写对其功能扩展。重写，重载。

重写是子类方法覆盖父类方法，要求方法名、返回值、参数相同；重载一个类中两个或以上的方法，只是方法名相同，返回值、参数都可不一样。

继承和实现

继承是子类对父类属性及行为的访问和扩展，继承后可以不使用父类继承的方法或是重写父类方法；实现必须实现接口中所定义的所有方法。

继承只能存在一个父类，实现接口可以同时实现多个接口，更加方便。

同步请求，异步请求

同步请求：发送方发送完数据，等待接收方回应后才能发下一个数据请求。

异步请求：发送方发送完数据，不需要等待接收方回应就可以发送下一个数据。

反射就是动态加载对象，并对对象进行剖析，对于任何一个类，根据类创建的实例或直接使用类来获取并使用这个类所有的属性和方法。

String，StringBuffer，StringBuilder

String是用final修饰符修饰的，final不能被继承，属于常量，创建后不能修改。少量字符串使用可以用”+”,每次+都会重新创建String对象，经常会使用Strngbuffer和Stringbuilder创建的字符串对象进行修改(底层也是这样实现的)。

StringBuffer是线程安全(也是被final修饰的)，单线程使用，慢一点；Stringbuilder线程不安全，线程安全时使用，快一点。

final类修饰

final用于不希望被继承的或者不允许被其他人操作的数据。

垃圾回收器

String s;s就是句柄。

它会查找用new创建的所有对象，辨别其中那些不再被引用，句柄指向的内存空间丢失就会被清除，随后，自动释放这些占用的内存，为其他对象创建提供内存。

垃圾回收器知道释放哪些new出来分配到内存，像int等特殊内存清除有局限，java提供了finallize()方法，准备号释放对象，首先调用finalize()，只有在下次来及清楚的时候会清除。

collection接口

collection是最基本的集合接口，他的主要子接口是list，set。map没有继承collection接口。

list在保存时包含位置(角标)list可以出现重复的情况，所以更多是进行指定为的的增删改查的方法；

set中是不包含重复的元素，set主要的两个子接口hashSet，treeSet；treeSet同时可以排序的去重。

map是一种包含键值对应的关联容器(包含键值对的对象数组)，map也是不能重复的，常用的两个子接口hashMap，treeMap；treeMap是有序的存放。

抽象类和接口

抽象类是集合子类的通用特性的 。它不能被实例化，只能被用作子类的超类。抽象类是被用来创建继承子类的模板。

接口是抽象方法的集合。如果一个类实现了某个接口，那么它就继承了这个接口的抽象方法。如果实现了这个接口，那么就必须确保使用这些方法。接口只是一种形式，接口自身不能做任何事情。

有些基类多个实现，使用抽象类更为合适，不需要每个继承或实现的完全重写所有方法，子类可以根据自身需要选择实现。

抽象类在接口和实体类之间的一个桥梁，接口更多时候是行为的规范，继承需要全部实现。

mybatis工作原理

mybatis初始化，加载并解析mybatis全局配置文件，mybatis通过SqlSessionFactory从mybatis配置文件获得并sqlsession，SqlSession封装的是jdbc连接，再通过实例化获得mapper对象并运行映射的sql语句，完成对数据库的增删改查，最后关闭SqlSession将接口转换为Map或是指定的javaBean返回给事务层。

实现多线程的方式

继承Thread类，重写run方法，没有返回值的。

实现Runnable接口，重写run方法，没有返回值的。

通过创建线程池，线程池创建线程，能够完成后返回Object数据。

线程池

线程池执行任务，判断当前线程数是否小于配置的最大线程数，大于则判断任务队列是否未满，未满则加入队列中，满了会拒绝任务抛异常。

线程池参数配置：允许的最大线程数，空闲线程存活时间，线程超时时间，等待的任务队列数。

索引

使用：主键，和其他表做连接的字段，在条件查询中或函数中用到，orderBY、groupBy使用的字段。

不适用：记录数据少，列很少的表，经常进行数据库操作交互的表。

失效的情况：sql中使用到多个索引不同使用(一个做条件，另一个分组啥的)，or关联两个索引列，模糊查询的

单例模式

1，饿汉式：私有化构造函数，私有化静态(static)修饰本对象的属性并new出来，对外提供的方法直接返回这个对象属性。

2，懒汉式：私有化构造函数，私有化本对象的属性，对外提供的方法要静态(static)修饰符修饰确保唯一，内部判断私有化的对象属性是否为空，为空才创建新对象。

3，静态内部类：私有化构造函数，私有化本对象的属性，创建一个新的静态内部类，类内部用final修饰一个静态(static)的属性(单例对象)的实例用new创建，对外提供一个公共方法return内部类的属性。
